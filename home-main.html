Accessibility in HTML with Examples & Best Practices

Purpose

Web accessibility is guided by four key principles: perceivable, operable, understandable, and robust. These principles ensure that all users, regardless of their abilities or disabilities, can access and interact with online content effectively. The purpose of the document "Accessibility in HTML with Examples and Best Practices" is to guide developers and content creators in designing accessible HTML components. It provides practical examples and best practices to ensure digital content is usable by everyone.

Web Complaince

Web accessibility compliance involves adhering to legal standards and guidelines that ensure digital content is accessible to all users, including those with disabilities. Laws like the Section 508 mandate specific accessibility criteria, while the Web Content Accessibility Guidelines (WCAG) outline best practices for creating accessible content. Compliance helps organizations avoid legal issues and fosters inclusivity, enhancing the user experience for a diverse audience.

Note*: The following documentation does not cover all of the WCAG AAA requirements. For all compliance, you can visit the provided links given in front of all four major principles: Perceivable, Operable, Understandable, and Robust.

  • Guideline 1.2 - Time-based Media requires that alternatives are provided for all time-based media (audio-only, video-only, audio-video, audio and/or video).
    • 1.2.1 Audio-only and Video-only (Prerecorded) requires text or audio alternatives for audio-only and video-only content, ensuring the same information is accessible.
      • 1.2.1-audio-transcript-text requires transcripts are provided for pre-recorded audio-only content. Transcripts must be accurate and complete.
      • 1.2.1-video- alternative-equivalent requires text or audio alternatives for prerecorded video-only content, conveying the same information as the video.
    • 1.2.2 Captions (Prerecorded) requires synchronized captions for all prerecorded audio content, ensuring accessibility for users.
      • 1.2.2-captions-equivalent requires that accurate captions are provided for pre-recorded synchronized media. Captions must be time-synchronized and equivalent.
    • 1.2.3 Audio Description or Media Alternative (Prerecorded) requires that prerecorded media content includes audio descriptions or alternative text tracks.
    • 1.2.4 Captions (Live) requires accurate captions for live synchronized media. The captions must be time-synchronized and equivalent.
  • 1.3 Adaptable ensures that the content can be presented in different ways without losing meaning.
    • 1.3.1 Info and Relationships ensures that information and its connections are programmatically identifiable, allowing assistive technologies to convey structure accurately.
      • 1.3.1-heading-determinable and levels ensures that headings are used correctly and in a hierarchical manner, allowing users to easily understand the content.
      • 1.3.1-list-type requires that all visually apparent lists are programmatically identified as the correct list type: ordered, unordered, or description list.
      • 1.3.1-programmatic-label requires that all form elements have programmatic associations which accurately describe the form element and include all relevant instructions.
      • 1.3.1-table-identification requires that data tables have the correct programmatic table markup to identify them as data tables.
      • 1.3.1-cell-header-association requires that data table cells are correctly associated with the relevant headers.
      • 1.3.1-layout-table-structure requires that layout tables are not coded as data tables. No data table semantics should be detected.
    • 1.3.2 Meaningful Sequence requires that content is presented in a logical order, ensuring that users can understand and navigate the information effectively.
      • 1.3.2-content-order-meaning-CSS- position ensures that the visual layout does not change the meaning of content, allowing assistive technologies to interpret it correctly.
    • 1.3.3 Sensory Characteristics requires that instructions for user interactions do not rely solely on sensory characteristics, such as shape, color, or sound.
    • 1.3.4 Orientation mandates that content must be presented in a way that supports both portrait and landscape orientations.
    • 1.3.5 Identify Input Purpose requires that users are informed about the expected purpose of input fields through clear labels or instructions.
  • 1.4 Distinguishable ensures users can easily perceive and differentiate content, emphasizing color contrast, text spacing, and visibility to enhance accessibility.
    • 1.4.1 Use of Color requires that color is not the only visual means used to convey information, indicate an action, prompt a response, or distinguish a visual element.
    • 1.4.2 Audio Control requires a method for pausing and stopping audio or controlling the volume of auto-playing content within the first three elements a user encounters on a web page.
    • 1.4.3 Contrast (Minimum) ensures that text and interactive elements have sufficient contrast against their background to enhance readability for users with visual impairments.
    • 1.4.4 Resize Text requires that there is a mechanism to resize, scale, or zoom in on the text to at least 200% of its original size without loss of content or functionality.
    • 1.4.5 Images of Text requires that images of text are either essential, such as logotypes, or presented along with user controls to customize font, size, color and background.
    • 1.4.10 Reflow ensures that content can adapt to various screen sizes, with a minimum width of 320 pixels and a height of 256 pixels.
    • 1.4.11 Non-text Contrast requires that visual elements, such as user interface components and graphical objects, have a contrast ratio of at least 3:1 against adjacent colors
    • 1.4.12 Text Spacing ensures that users can adjust the spacing between characters, words, lines, and paragraphs without loss of content or functionality.
    • 1.4.13 Content on Hover or Focus requires that any content displayed on hover or focus (like tooltips or menus) must be accessible and not obscure other content.
