Sensory Characteristics

Fig 1 Usage in HTML:

According to the Web Content Accessibility Guidelines (WCAG), content should not rely on a single sensory characteristic, such as color, to convey important information. This is particularly critical for users with visual impairments, such as color blindness, who may not be able to perceive information presented solely in color.

In the case of our task completion table, we use both color and text to convey the status of each task: green indicates "Completed," and orange indicates "In Progress." This approach ensures that all users, regardless of their ability to perceive color, can understand the status of the tasks. If we were to rely only on color without accompanying text labels, users who cannot distinguish between the colors would not be able to understand the information, thus failing the sensory characteristics guideline. Therefore, when designing content that conveys important information, it is essential to combine multiple sensory cues—such as text, color, and icons—so that all users can access the information effectively. 

              
<!DOCTYPE html>
<html lang="en">
<head>
  <title>WCAG Sensory Characteristics Demo</title>
  <style>
    body {
        font-family: Arial, sans-serif;
        margin: 20px;
    }
    table {
        border-collapse: collapse;
        width: 100%;
        margin-top: 20px;
    }
    th, td {
        border: 1px solid #ccc;
        padding: 10px;
        text-align: center;
    }
    th {
        background-color: #f4f4f4;
    }
    .status-1 {
        background-color: #4CAF50; /* Green */
        color: white;
    }
    .status-2 {
        background-color: #FF9800; /* Orange */
        color: white;
    }
    .visually-hidden {
        position: absolute;
        width: 1px;
        height: 1px;
        margin: -1px;
        clip: rect(0, 0, 0, 0);
        overflow: hidden;
    }
  </style>
</head>
<body>
  <h1>WCAG Sensory Characteristics Demo</h1>
  <table>
    <thead>
        <tr>
            <th>Item</th>
            <th>Status</th>
        </tr>
    </thead>
    <tbody>
      <tr>
        <td>Task 1</td>
        <td class="status-1">
          <span class="visually-hidden">Completed</span>
          ✓ Completed
        </td>
      </tr>
      <tr>
        <td>Task 2</td>
        <td class="status-2">
          <span class="visually-hidden">In Progress</span>
          ⚠ In Progress
        </td>
      </tr>
    </tbody>
  </table>
  <p><strong>Note:</strong> Green indicates completed tasks, and orange indicates tasks that are in progress.</p>
</body>
</html>

This interface includes two arrow-shaped buttons, one for moving to the next section ("Next") and another for moving to the previous section ("Previous"). The buttons are designed with arrows, which are a visual cue, but important information is also provided in text form. This ensures compliance with the WCAG Sensory Characteristics guideline, which states that information should not rely solely on one sensory characteristic, such as color or shape.

In this case, the blue arrow icons are accompanied by text labels ("Next" and "Previous") to provide clear, multi-sensory information. If the arrow icons were used without text, users who rely on assistive technologies (such as screen readers) or those with visual impairments might not understand the function of the buttons. The use of both visual (color and shape) and textual information allows users to navigate the page effectively, regardless of their sensory abilities.
